# Gitignore Requirements

## Scope

Applies to repository ignore policy and generated local artifacts configured in `.gitignore` for this project.

R001  Statement: Exclude local build output from version control.
Design: Ignore repository-local build directory `.build/` recursively.
Rationale: Build artifacts are machine-generated, high-churn, and dramatically increase push size.
Tests:
- R001-T01: New `.build/` build output is git-ignored.

R005  Statement: Exclude Xcode DerivedData and user-specific metadata from version control.
Design: Ignore `DerivedData/`, `*.xcuserdatad/`, and `*.xcuserstate`.
Rationale: These paths are local IDE state and are not portable across machines.
Tests:
- R005-T01: Xcode DerivedData and user-specific metadata are git-ignored.

R010  Statement: Keep project source and shared project configuration tracked.
Constraints:
- Do not ignore `macos_app/` source files.
- Do not ignore `macos_app/OutlookMailApp.xcodeproj/project.pbxproj`.
- Do not ignore `macos_app/OutlookMailApp.xcodeproj/project.xcworkspace/contents.xcworkspacedata`.
Tests:
- R010-T01: Project source and shared project configuration stay tracked, not ignored.

R015  Statement: Remove accidentally tracked generated artifacts from Git index without deleting local files.
Design: Use cached removals (`git rm -r --cached`) for ignored artifact paths.
Rationale: Keeps local developer caches intact while cleaning repository history moving forward.
Tests:
- R015-T01: Cached-removal cleanup keeps ignored artifacts untracked while preserving local files.

R020  Statement: Enforce that ignored artifact paths are not tracked after cleanup.
Design: Verify `git ls-files` contains no entries under ignored build/cache paths.
Tests:
- R020-T01: `git ls-files` reports no entries under ignored build/cache paths.

R025  Statement: Exclude project virtual environment directories from version control.
Design: Ignore local virtualenv directories (`.venv/` and `*-venv/`) used for Python dependency installation and execution.
Rationale: Virtual environments are machine-specific and can contain large binary artifacts.
Tests:
- R025-T01: Local virtualenv directories are git-ignored.

R030  Statement: Exclude generated dependency freshness reports from version control.
Design: Ignore `reports/dependency-freshness/` outputs generated by `tests/t02_run_dependency_freshness_tests.sh`.
Rationale: Freshness reports are ephemeral local diagnostics that should not churn repository history.
Tests:
- R030-T01: Generated dependency freshness reports are git-ignored.

R035  Statement: Exclude LLVM raw profile artifacts from version control.
Design: Ignore `default.profraw` generated by local Xcode/Swift profile-instrumented runs.
Rationale: Profiling raw outputs are transient local diagnostics and should not be committed.
Tests:
- R035-T01: `default.profraw` profiler artifact is git-ignored.

R040  Statement: Exclude Python bytecode cache directories from version control.
Design: Ignore `__pycache__/` recursively wherever Python modules are imported or executed.
Rationale: Bytecode caches are machine-generated and recreated automatically by the interpreter.
Tests:
- R040-T01: Python `__pycache__/` directories are git-ignored.

R045  Statement: Exclude standalone Python bytecode artifacts from version control.
Design: Ignore compiled bytecode files (`*.pyc`) that may be emitted outside `__pycache__/` directories.
Rationale: Bytecode artifacts are generated outputs and should not be tracked in source control.
Tests:
- R045-T01: Standalone Python `*.pyc` bytecode files are git-ignored.
